Section 01
What is Euclidean Distance?
d(A,B) = √( Σ (aᵢ − bᵢ)² )
sum over all dimensions i · square root of squared differences

The straight-line distance between two points in n-dimensional space — the classic "as the crow flies" measurement. In 2D: √((x₂−x₁)² + (y₂−y₁)²). Generalizes naturally to any number of dimensions.

Works perfectly for physical measurements — GPS coordinates, pixel colors, sensor readings. But breaks down for NLP because of the magnitude problem: two documents about the same topic can be far apart if one is longer.

Section 02
The Magnitude Problem
PROBLEM — Euclidean

Length skews distance

A short and long article on the same topic produce vectors pointing in the same direction — but with different magnitudes. Euclidean measures the gap between tips, so the long doc appears dissimilar.

SOLUTION — Cosine

Direction only, not length

Cosine measures the angle between vectors. Same direction = angle 0° = similarity 1.0, regardless of length. Topics cluster correctly; document length no longer pollutes the score.

Both vectors point in identical direction (45°) — same topic.
Euclidean distance ≈ 4.24. Cosine similarity = 1.0.

Section 04
Key Takeaway
// foundational insight In NLP, embedding models often normalize output vectors to unit length before storing them. When vectors are unit-normalized, Euclidean distance becomes mathematically proportional to cosine distance — so the distinction disappears at inference time.

But understanding why cosine is preferred for raw embeddings is foundational: it makes similarity magnitude-invariant, so a long document and a short document on the same topic receive the same score. Without this, vector search would be biased toward documents of similar length rather than similar meaning.
